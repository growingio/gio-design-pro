# PastTimeSelector 过去时间选择器

<Subtitle>用户可以通过日期区间选择器选择想要选择的相对和绝对的日期区间。</Subtitle>

## 代码演示

### 样式 - 「常用时间」

- 【1】左侧 List：用户使用左侧 List 选择日期区间的计算方式，该 List 应继承通常 List 的交互样式；
- 【2】右侧 List 为常用时间区间，样式为双排 List；
- 【3】当用户选择完成常用时间，时间按钮文案直接显示选择的选项。

### 样式 - 「从某日至今」

当用户选择“从某日至今”时，用户可以借助时间选择器选择从某一特定日期至今日的动态区间（今日日期为动态）。由于只有开始日期需要选择，我们让用户通过更改日历来做选出开始日期。

- 【1】用户需通过输入或日历选择开始日期，日历上显示的区间为开始日期到今日，用户也可从今日切换到昨日。
- 【2】由于今日/昨日无法改变，所以右边的日历锁定在当前月份，右边日历所有按钮为 disabled 状态。
- 【3】选择完成后，按钮文案格式为“自 YYYY/MM/DD 至今“。

### 样式 - 「动态时段」

当用户选择“过去 N 天”时，用户可以借助时间选择器选择过去 N 天或者过去 N~M 天，这样他可以查看（相对今天）过去一段时间范围的数据。我们让用户用过输入数字或更改日历来做出 N 天的选择。

- 【1】当用户选择“过去 N 天”选项，初始值为 7 天。用户可通过选改数字或日历更改“N 天”。选改后，数字或日历应做出呼应。
- 【2】用户可通过修改日历上的开始日期来设定 N 值。用户只可修改开始日期，不可在日历上修改结束日期。结束日期默认为今日。N 为非负整数，最大值为 10000.
- 【3】若用户点击“结束日期”，则会出现【4】
- 【4】当用户选择有“结束日期”，则可选择过去 N-M 天，这里默认 N 为 0，而 M 为之前选择的数字。日历与数字相互回应。
- 【5】根据用户选择，按钮文案为“过去 N 天”或“过去 N-M 天“

### 样式 - 「过去固定时段」

当用户选择“过去固定时段”时，用户可以借助时间选择器选择日期区间的开始日期和结束日期，该日期区间为固定日期区间。

- 【1】用户通过输入或日历选择开始日期与结束日期，该交互模式与“日期选择器 Date Picker”规范相同。
- 【2】选择完成后，按钮文案格式为“YYYY/MM/DD 至 YYYY/MM/DD“。
